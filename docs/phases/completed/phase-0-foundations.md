### 0.7 Security Headers in `hooks.server.ts`

Add security headers to all responses via the `handleBetterAuth` hook (or a dedicated `handleSecurityHeaders` hook):

- `Content-Security-Policy` ‚Äî restrictive baseline (relaxed incrementally per route)
- `Strict-Transport-Security` ‚Äî `max-age=63072000; includeSubDomains; preload`
- `X-Content-Type-Options` ‚Äî `nosniff`
- `Referrer-Policy` ‚Äî `strict-origin-when-cross-origin`
- `Permissions-Policy` ‚Äî `camera=(), microphone=(), geolocation=()`

Also add route guards for `/app/*` and `/admin/*`:

- `/app/*` ‚Äî redirect to `/login?next=<path>` if no session
- `/admin/*` ‚Äî redirect to `/login?next=<path>` if no session; return 403 if session but not admin

**Files:**

- `src/hooks.server.ts` ‚Äî add `handleSecurityHeaders` + `handleRouteGuards` to sequence
