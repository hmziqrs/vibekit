# Auth & Security Audit — 2026-05-15

## Methodology

Each claimed feature from `docs/loop.md` (Auth & Security section, lines 93-104) was verified by reading source files, tracing call chains, and checking for end-to-end completeness. Statuses:

- **COMPLETE**: Fully wired end-to-end (config, backend, UI, tests)
- **PARTIAL**: Core mechanism exists but has gaps or missing sub-features
- **NOT IMPLEMENTED**: No code found for the claimed feature
- **NON-FUNCTIONAL**: Code exists but cannot work in its current form

## Critical Gaps

1. ~~**No Content-Security-Policy header**~~ — **FIXED**. `handleSecurityHeaders` now generates a nonce per request via `crypto.randomUUID()`, sets `Content-Security-Policy` with nonce-based `script-src`, and makes it available via `event.locals.cspNonce`. CSP covers: default-src, script-src (nonce + Cloudflare), style-src (unsafe-inline for Svelte), img-src, connect-src (Stripe), frame-src (Stripe/Cloudflare), form-action, base-uri, object-src.

2. ~~**Rate limiting uses in-memory storage — ineffective in production**~~ — **DONE**. Switched to D1-backed rate limiting: `rate_limit_log` table in schema, `dbRateLimitCheck()` in `rate-limit.ts` with D1 queries. Better Auth rate limiter switched to `storage: 'database'`. Falls back to in-memory for tests/Node adapter.

3. ~~**HaveIBeenPwned breached password check is completely missing**~~ **DONE** -- HIBP k-anonymity check implemented in `src/lib/server/security/hibp.ts`.

4. ~~**No security alert emails**~~ — **FIXED**. Added `sendSecurityAlert()` method to `EmailService` with `security-alert.ts` template covering: account_locked, new_device, password_change, two_factor_change. Hooks fire alerts after each event type. Emails are sent fire-and-forget (don't block the response).

5. ~~**No concurrent session limits**~~ **DONE** -- Max 5 concurrent sessions enforced in `hooks.server.ts`, oldest evicted on new login.

6. ~~**Session IP/UA may not be populated on creation**~~ — **DONE**. IP and user agent now populated on the latest session during login in `hooks.server.ts`.

7. ~~**No 2FA enforcement per role or organization**~~ — **DONE**. Admin users are now required to have 2FA enabled. Redirects to `/app/settings?require2fa=1` with prominent banner.

8. ~~**"Remember device" for 2FA is not implemented**~~ — **DONE**. Better Auth's built-in `trustDevice` parameter enabled in `twoFactor` plugin with `trustDeviceMaxAge: 30 days`. The 2FA verification page has a "Trust this device for 30 days" checkbox. Trusted device management API (`GET/DELETE /trusted-devices`) for listing/revoking. Custom `trusted_device` table for admin management.

9. ~~**Progressive backoff is not implemented**~~ — **FIXED**. `auth-lockout.ts` now uses exponential backoff: base 5 minutes, doubling each time (`BASE_LOCKOUT_MS * Math.pow(2, newCount - MAX_ATTEMPTS)`). First lockout = 5 min, second = 10 min, third = 20 min, etc. Cap is effectively unlimited (the window resets after 15 min of no attempts).

10. ~~**Action quotas per subscription tier do not exist**~~ — **DONE**. Tiered rate limit foundation added in `middleware.ts` with `TIER_MULTIPLIERS` for pro/team/enterprise plans.

11. ~~**Ban evasion detection via email/IP is not implemented**~~ — **DONE**. Ban evasion detection added in `src/lib/server/security/ban-evasion.ts`, runs on user registration via `auth.ts` hook, flags via audit log.
