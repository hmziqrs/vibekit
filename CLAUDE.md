## Project Overview

Vibekit is a SvelteKit SaaS boilerplate targeting Cloudflare Workers. Stack: Svelte 5 (runes mode), SvelteKit 2, Cloudflare D1 (SQLite), Drizzle ORM, Better Auth, Tailwind CSS v4, shadcn-svelte, Paraglide JS (i18n), TanStack libraries. Package manager is Bun.

## Commands

```bash
bun run dev                    # Start dev server
bun run build                  # Type-check wrangler types + production build
bun run preview                # Preview production build locally on port 4173
bun run check                  # Full type checking (wrangler types + svelte-check)
bun run gen                    # Generate Cloudflare Workers types

# Database
bun run db:generate            # Generate migration SQL from schema changes
bun run db:push:local          # Push latest migration to local D1
bun run db:push                # Push schema to remote D1 (needs Cloudflare creds)
bun run db:migrate:local       # Run all local migrations sequentially
bun run db:studio:local        # Visual DB browser (local)
bun run db:studio              # Visual DB browser (production)
bun run auth:schema            # Regenerate auth schema from Better Auth config

# Quality
bun run lint                   # oxlint
bun run lint:fix               # oxlint --fix
bun run format                 # oxfmt --write
bun run format:check           # oxfmt --check
bun run test                   # vitest
bun run test -- path/to.test.ts  # Run a single test file

# Adding shadcn-svelte components
bunx shadcn-svelte@latest add <component>
```

## Architecture

### Rendering Strategy
- **Landing page** (`/`): Prerendered static (`prerender = true`, `csr = false`)
- **App routes** (planned): CSR-only (`ssr = false`) behind auth, data via TanStack Query
- **Blog** (planned): SSR at edge with Cloudflare Cache API

### Svelte 5 Runes
Runes mode is forced on globally via `svelte.config.js`. Always use `$props()`, `$state()`, `$derived()`, `$effect()` — never legacy Svelte 4 syntax (`export let`, `$:`, `on:` directives, etc.).

### Auth Pattern
Auth uses a **factory function** `createAuth(d1)` because D1 is only available per-request in Cloudflare Workers. Access auth via `event.locals.auth` in server code. The auth instance and session/user are set in `hooks.server.ts`. Auth endpoints are handled automatically by Better Auth's `svelteKitHandler`.

Server-side auth guard pattern: check `event.locals.user` in `+page.server.ts` load functions and redirect as needed.

### Database
- Schema: `src/lib/server/db/schema.ts` (app tables) + `auth.schema.ts` (auto-generated by Better Auth)
- Client factory: `getDb(d1)` in `src/lib/server/db/index.ts`
- IDs: UUID v7 via `uuidv7()` package, wrapped in `src/lib/server/uuid.ts`
- D1 binding is `event.platform.env.DB` (configured in `wrangler.jsonc`)
- `src/lib/server/` modules must never be imported from client code

### Server Hooks
`hooks.server.ts` uses `sequence(handleParaglide, handleBetterAuth)`. i18n middleware runs first, then auth. Both gracefully skip during build/prerender when DB is unavailable.

### i18n
Paraglide JS with compiled output in `src/lib/paraglide/` (auto-generated, do not edit). Source messages in `messages/en.json` and `messages/ur.json`. Supports English and Urdu. Locale is URL-based; HTML lang/dir attributes are set via template replacement in hooks.

### Styling
Tailwind CSS v4 with shadcn-svelte theme tokens defined as CSS custom properties in `src/routes/+layout.css`. Dark mode by default (`class="dark"` on `<html>`). Use `cn()` from `$lib/utils` for conditional class merging.

### Code Style
- oxlint with typescript/unicorn/import/vitest plugins
- oxfmt: single quotes, no semicolons, 100 char width, sorted imports, trailing commas (es5)
- Tab width: 2 spaces

### Environment Variables
Local dev only needs `ORIGIN` and `BETTER_AUTH_SECRET` in `.env`. Cloudflare D1 credentials (`CLOUDFLARE_ACCOUNT_ID`, `CLOUDFLARE_DATABASE_ID`, `CLOUDFLARE_D1_TOKEN`) are only needed for remote database operations.
